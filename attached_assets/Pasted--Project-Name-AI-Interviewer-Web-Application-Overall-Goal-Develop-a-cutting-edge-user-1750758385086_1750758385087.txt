

**Project Name:** AI Interviewer Web Application



**Overall Goal:** Develop a cutting-edge, user-friendly AI Interviewer web application that provides a realistic, multi-stage, and highly effective mock interview experience. The application must leverage AI for dynamic question generation, sophisticated response evaluation, and insightful, real-time feedback (including voice). The UI should be modern, intuitive, and visually appealing, leaving users impressed by both its aesthetics and functionality. The entire interview experience, from user onboarding to detailed feedback, should be **excellent** with all functionalities working correctly and seamlessly.



**Target Audience:** Job seekers, students, and professionals looking to practice and improve their interview skills across various domains and experience levels.

### **1. Core Functionality (What the App Does)**



The application will provide the following key features:



* **User Authentication:** Secure user registration and login (email/password, social login options).

* **Profile Management:** Users can create and manage their profiles, including selecting job roles/industries of interest, experience levels, and preferred interview languages.

* **User Flow Post-Registration:** After account creation, the candidate will be directly navigated to a "Home" or "Dashboard" page, immediately presenting options to select a role and start an interview.

* **Multi-Stage Mock Interview Simulation:** Users select a job role/industry and experience level. The interview will proceed through three distinct stages:



    1.  **Stage 1: MCQ Questions**

        * **Question Generation:** Gemini AI will automatically generate a set of multiple-choice questions relevant to the selected role/experience (e.g., technical MCQs for tech roles, general knowledge for non-tech).

        * **Answer Evaluation:** User-selected MCQ answers will be evaluated by Gemini AI for correctness, providing immediate feedback.

    2.  **Stage 2: Coding Test (Conditional)**

        * **Mandatory for Technical Roles:** If the selected role is technical (e.g., Software Engineer, Data Scientist, Web Developer), this stage is automatically included.

        * **Optional/Skipped for Non-Technical Roles:** If the selected role is non-technical (e.g., Marketing Manager, HR Specialist, Sales), this stage is skipped or presented as optional.

        * **Question Generation:** Gemini AI will generate coding problems relevant to the specific technical role and chosen experience level.

        * **Code Input:** Users will write and submit their code in an integrated, syntax-highlighted code editor.

        * **Code Evaluation:** **Judge0 API** will be used to compile, run, and evaluate the submitted code against predefined or dynamically generated test cases (e.g., correctness, time complexity, memory usage).

        * **Feedback:** Real-time feedback on code performance (e.g., "Tests Passed," "Runtime Error," "Wrong Answer") will be displayed.

    3.  **Stage 3: Real-time Viva Voice Interview**

        * **AI Behavior:** The AI will behave as a realistic HR interviewer, simulating a natural conversation flow.

        * **Voice-over Questions (AI to User):** The AI will ask interview questions aloud using Text-to-Speech (TTS) capabilities (either Gemini's built-in TTS or a dedicated TTS API like Google Cloud Text-to-Speech).

        * **Text Display of Questions:** The AI's spoken question will simultaneously be displayed in text on the screen for user reference and accessibility.

        * **Dual Answer Input (User to AI):** Users can respond to the AI's question via:

            * **Text Input:** A standard multi-line text area.

            * **Audio Recording:** A microphone input to record their voice.

        * **Audio Transcription:** The recorded audio response from the user will be transcribed into text using the **OpenAI Whisper API**.

        * **AI Evaluation & Real-time Feedback:** Gemini AI will evaluate the transcribed text (and potentially the content from the text input field) for relevance, clarity, communication skills, and confidence cues.

        * **Voice-over Suggestions (AI to User):** Crucially, after *every question*, the AI will provide constructive suggestions and insightful feedback *through voice-over*, mimicking a real HR interviewer's guidance. This feedback will also be displayed in text.

* **Interview Completion & Comprehensive Feedback:**

    * After all three stages are completed, the interview concludes.

    * AI will generate a detailed feedback report.

    * The report will clearly mention **strengths and weaknesses** observed across all stages.

    * It will rate key soft skills such as **confidence, communication skills, and overall improvement potential**.

    * Finally, an overall **score** for the interview will be provided.

* **Historical Interview Data:** Users can view summaries and detailed reports of all past mock interviews.

* **Responsive Design:** The application must be fully responsive, working seamlessly on desktop, tablet, and mobile devices.

### **2. Dependencies & Tech Stack (Detailed)**



* **Frontend Framework:** React.js

* **Styling:** Tailwind CSS (for rapid, utility-first styling and easy responsiveness)

* **State Management:** React Context API or Redux (Cursor can suggest based on complexity, but prefer Context for initial simplicity)

* **Backend (API Layer):** Flask (Python-based RESTful APIs)

* **Database:** *Cursor will decide based on the project requirements and suggest a suitable database (e.g., MongoDB for flexibility, PostgreSQL for relational data).*

* **Authentication:** Firebase Authentication

* **AI & External API Integration:**

    * **Primary AI Model:** Google Gemini API (for question generation, MCQ evaluation, coding problem generation, Viva Voice answer evaluation, detailed feedback generation, and potentially AI voice output).

    * **Speech-to-Text (User Response):** OpenAI Whisper API (for transcribing user's audio responses).

    * **Coding Evaluation:** Judge0 API (for running and evaluating user-submitted code).

    * **Text-to-Speech (AI Questions/Feedback):** Utilize Gemini's built-in TTS capabilities or suggest integration with a dedicated TTS service (e.g., Google Cloud Text-to-Speech, ElevenLabs) if more advanced voice control or specific voice characteristics are desired.

* **Deployment (for later, but keep in mind during architecture):** Vercel/Netlify (for React frontend), Render/AWS EC2 (for Flask backend).

### **3. General UI/UX Principles (To impress with the design)**



* **Clean and Modern Aesthetic:** Minimalist design, ample whitespace, clear typography, and a balanced color palette.

* **Intuitive Navigation:** Easy-to-understand menus, clear calls to action, and logical user flows.

* **Visual Hierarchy:** Use size, color, and placement to guide the user's eye to the most important information.

* **Consistency:** Uniformity in design elements (buttons, forms, cards, typography, colors) across all pages.

* **Feedback and Micro-interactions:** Subtle animations, hover effects, and clear loading states to enhance user experience and provide immediate feedback, especially during AI processing.

* **Accessibility:** Adhere to WCAG guidelines for color contrast, keyboard navigation, and screen reader compatibility.

* **Engaging and Professional:** The design should feel professional and encouraging, reducing interview anxiety rather than increasing it. The overall look and feel should inspire confidence and trust in the AI's capabilities.
